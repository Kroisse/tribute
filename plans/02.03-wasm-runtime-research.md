# WebAssembly 런타임 조사 (WasmGC + WASI)

> 조사일: 2025-12-18

Tribute 컴파일러의 WasmGC 백엔드 개발을 위해 주요 WebAssembly 런타임들의 WasmGC 및 WASI 지원 현황을 조사함.

## 주요 런타임 비교

### 1. Wasmtime (추천)

- **WasmGC**: 2024년 11월 버전 27.0부터 완전 지원
- **WASI**: 0.2 완전 지원 (Component Model 포함), 0.3은 2025년 상반기 예정
- Bytecode Alliance에서 개발하며, 가장 적극적으로 새 proposal들을 구현
- 현재 버전 35.0까지 출시됨
- Cranelift을 JIT/AOT 백엔드로 사용

**장점**:

- WasmGC + WASI 모두 완전 지원
- Tribute가 이미 Cranelift 백엔드를 사용하므로 생태계 일관성
- Component Model 지원으로 모듈 간 상호운용성 확보

### 2. WasmEdge

- **WasmGC**: 인터프리터 모드에서만 지원 (`--enable-gc` 플래그)
- **WASI**: 지원
- CNCF 프로젝트로, 클라우드 네이티브/엣지 환경에 최적화
- AOT 컴파일된 GC 지원은 아직 제한적

**장점**:

- 클라우드 네이티브 환경에 최적화
- 경량 런타임

**단점**:

- WasmGC가 인터프리터 전용으로 성능 제한

### 3. Wasmer

- **WasmGC**: 아직 미지원 (GitHub issue #357에서 논의 중)
- **WASI**: 지원
- 사용하기 쉬운 API가 장점

**단점**:

- WasmGC 미지원으로 Tribute 타겟으로 부적합

### 4. 브라우저 (V8, SpiderMonkey, JavaScriptCore)

- **WasmGC**: 2024년 12월 기준 모든 주요 브라우저에서 기본 활성화
- 브라우저 환경에서는 완벽히 지원됨

## WebAssembly 생태계 동향

### WASI 로드맵

- **WASI 0.2**: 현재 안정 버전, Component Model 포함
- **WASI 0.3**: 2025년 상반기 예정, 네이티브 비동기 지원 추가

### WebAssembly 3.0

- 2025년 9월 표준화 목표
- 주요 기능: Wasm64 (4GB 이상 메모리), 스레딩/SIMD 개선

## 결론

**Tribute 프로젝트에는 Wasmtime을 기본 타겟 런타임으로 권장**:

1. WasmGC + WASI 완전 지원
2. Cranelift 기반으로 Tribute 백엔드와 생태계 일관성
3. Bytecode Alliance의 적극적인 개발 및 유지보수
4. Component Model 지원으로 향후 모듈 시스템 확장 용이
